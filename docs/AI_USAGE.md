# AI use and human control

AI was used as a production collaborator, not as an unchecked runtime authority.

## Where AI contributed

- Turning human product intent into a structured experience seed.
- Proposing and reviewing world architecture.
- Drafting narrative material inside a fixed graph and schema.
- Helping implement compilers, validators and runtime adapters.
- Generating visual assets for the historical dossier.
- Finding regressions and converting repeated failures into automated QA.

## Where deterministic controls take over

- Node and action identifiers.
- State transitions.
- Resource acquisition and consumption.
- Causal promises and ending requirements.
- Reachability and route enumeration.
- Runtime schema adaptation.
- Visual bindings.
- Mobile and HTTP smoke checks.

The submitted guided playthrough performs no model calls. This makes the judged experience reproducible even without credentials or network access.

## Human feedback loop

The workflow preserves three human checkpoints:

1. Confirm the intended experience before world architecture.
2. Approve or reject structural and visual outputs.
3. Record recurring failures as contracts or executable QA instead of relying on longer prompts.

The in-app **Cambiar texto** control also records a local editorial correction. Those corrections remain local and are excluded from the repository.

## Safety and privacy

- No API key is included.
- Raw conversations and local logs are excluded.
- Raw personal reference images are excluded.
- Published character names and faces are fictional.
- Historical places are kept because locality is part of the creative premise.
