# Submission QA

This folder contains the executable release gates retained for the Project DM vertical slice.

The entry point is:

```bash
npm test
```

It validates, in order:

1. Canonical schema and visible narrative.
2. Agency, path expansion and reachable endings.
3. Resource economy.
4. Causal persistence.
5. Runtime adaptation.
6. Experience signature.
7. Mobile layout.
8. Visual manifest and coverage.
9. Guided HTTP playthroughs.

The suite is intentionally world-specific for the public submission. Broader internal factory experiments and QA for unpublished worlds are not included.
